以下の指示に従ってPull Requestのタイトルと説明を生成してください：

1. タイトルについて:
   - 簡潔で明確なタイトルを付けてください（50文字以内が理想）
   - タイトルの言語は英語
   - タイトルの最初は小文字
   - タイトルの形式: '[タイプ]: 変更内容の要約'
   - タイプの例: fix, feat, refactor, docs, test, chore など

2. 説明について:
   - 基本はWhatとWhyのセクションをシンプルに記載してください
   - 変更点は箇条書きでリストアップしてください (リストの数は多くなりすぎないように、2,3個がベスト)
   - 破壊的変更がある場合は目立つように記載してください
   - 関連するチケットやIssueがある場合はリンクしてください

3. スタイル:
   - 専門的かつ明確な言葉遣いを心がけてください
   - 長すぎる説明は避け、300-500文字程度に収めてください
   - 絵文字は適度に使用してOKです（特に見出しの前など）
